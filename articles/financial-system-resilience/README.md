# Financial System Resilience

Companion repository folder for the article **Financial System Resilience**.

This directory provides reproducible workflows for financial resilience strategy comparison, capital and liquidity buffer analysis, payment and clearing infrastructure review, governance and stress-testing diagnostics, non-bank financial exposure review, climate and cyber risk assessment, inclusion and household balance-sheet resilience, contagion simulation, and uncertainty-aware financial-stability planning.

## Correct article directory

```text
articles/financial-system-resilience/
```

## Purpose

This scaffold supports a professional financial system resilience workflow:

1. Define financial resilience strategies across capital strength, liquidity resilience, infrastructure robustness, governance capacity, inclusive resilience, systemic exposure, and implementation burden.
2. Compare strategies under balanced, capital-first, liquidity-first, infrastructure-first, governance-first, inclusion-first, exposure-sensitive, and implementation-aware priorities.
3. Simulate system stress under credit losses, liquidity runs, fire-sale contagion, cyber outage, climate repricing, non-bank financial stress, payment disruption, and compound financial instability.
4. Estimate inclusion-adjusted, infrastructure-adjusted, and exposure-adjusted financial resilience value.
5. Diagnose capital gaps, liquidity gaps, infrastructure gaps, inclusion gaps, systemic-exposure triggers, implementation burden, and operational resilience weaknesses.
6. Model uncertainty using Monte Carlo simulation.
7. Support responsible interpretation across financial stability, economic resilience, institutional resilience, payment systems, banking, non-bank finance, cyber risk, climate risk, household financial resilience, and public governance.
8. Provide multi-language examples in Python, R, Julia, SQL, Rust, Go, C, C++, and Fortran.
9. Export reproducible tables, figures, documentation, and notebook placeholders.

## Responsible-use note

The data are synthetic and the models are methodological examples. They are intended for learning, reproducible article companions, dashboard prototyping, financial resilience strategy comparison, stress-test workflow design, and professional methodology demonstration.

Real financial-system resilience analysis requires confidential supervisory data, institution-specific balance-sheet data, market data, payment-system data, liquidity data, counterparty exposure data, household and small-firm vulnerability data, climate-risk information, operational-risk evidence, legal review, institutional judgment, and accountable public governance.

Do not use this scaffold to automate financial supervision, rank institutions for real-world action, make investment decisions, infer institutional soundness from synthetic data, justify bailouts or austerity, obscure unequal financial harm behind aggregate metrics, or replace professional regulatory judgment.

## Quick start

Run the dependency-light Python workflow:

```bash
python3 python/financial_system_resilience_standard.py
```

Optional advanced Python workflow:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-advanced.txt
python python/financial_system_resilience_advanced.py
```

Run smoke checks:

```bash
bash smoke-test.sh
```
