# Modeling Principles

This companion scaffold models resilience indicators and dashboard risk.

## Core variables

- indicator coverage
- threshold sensitivity
- justice visibility
- uncertainty transparency
- decision-trigger clarity
- learning integration
- dashboard-risk exposure
- exposure reduction
- recovery capacity
- adaptive capacity
- buffer capacity
- threshold risk
- missingness
- red-flag conditions

## Core modeling logic

Dashboard value rises when indicator systems cover multiple resilience domains, detect threshold risk, show justice and distribution, label uncertainty, connect to decisions, and support learning. Dashboard value falls when false precision, hidden inequality, metric fixation, lagging visibility, and dashboard complacency increase.

The scaffold calculates:

1. dashboard design value under multiple priorities
2. naive dashboard scores
3. threshold-adjusted scores
4. uncertainty-adjusted scores
5. red-flag conditions that should not be averaged away
6. Monte Carlo uncertainty rankings
7. synthetic classifier outputs for high-value, low-dashboard-risk designs

## Interpretation

These examples are transparent modeling scaffolds, not empirical claims. Real dashboard design requires domain expertise, data governance, community participation, transparent assumptions, uncertainty labeling, and accountable decision triggers.
