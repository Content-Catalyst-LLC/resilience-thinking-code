# Resilience Thinking: Disturbance, Adaptation, and Thresholds

This folder supports the Resilience Thinking pillar with reproducible examples for:

- resilience-profile comparison
- repeated disturbance simulation
- adaptive capacity and threshold distance
- redundancy, diversity, modularity, and learning indicators
- threshold-risk modeling
- network resilience diagnostics
- adaptive-cycle documentation
- scenario comparison
- SQL resilience metadata

Important distinction: resilience is not automatically good. A system may be resilient while preserving injustice, ecological harm, exclusion, or coercion.
