# Transformation in Complex Systems

Companion repository folder for the article **“Transformation in Complex Systems.”**

This directory provides reproducible workflows for transformation pathway scoring, adaptive-limit diagnostics, governance readiness, justice contribution, ecological viability, structural-risk penalties, scenario-weighted decision support, and Monte Carlo uncertainty analysis.

## Correct article directory

```text
articles/transformation-in-complex-systems/
```

## Purpose

This scaffold supports a professional transformation analysis workflow:

1. Distinguish adaptation within a regime from transformation into a new regime.
2. Compare transformation pathways across adaptive support, transformability, governance readiness, justice contribution, ecological viability, legitimacy, resource feasibility, and structural risk.
3. Run weighted scenario comparisons under balanced, adaptation-first, transformation-first, governance-first, justice-first, ecological-viability-first, and risk-sensitive priorities.
4. Model uncertainty using Monte Carlo simulation.
5. Diagnose transformation readiness, pathway robustness, risk penalties, and possible transition failure modes.
6. Provide transparent decision-support artifacts for governance, resilience, public-interest planning, and social-ecological systems work.
7. Provide multi-language examples in Python, R, Julia, SQL, Rust, Go, C, C++, and Fortran.
8. Export reproducible tables, figures, documentation, and notebook placeholders.

## Responsible-use note

The data are synthetic and the models are methodological examples. They are intended for learning, reproducible article companions, dashboard prototyping, and professional modeling design. Real transformation decisions require public deliberation, local knowledge, affected-community authority, domain expertise, empirical evidence, legal review, and accountable governance.

Do not use these examples to impose transformation, rank communities, justify displacement, excuse austerity, or replace political judgment with a model score.

## Quick start

Run the dependency-light Python workflow:

```bash
python3 python/transformation_standard.py
```

Optional advanced Python workflow:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-advanced.txt
python python/transformation_advanced.py
```

Run smoke checks:

```bash
bash smoke-test.sh
```
