# Learning, Memory, and Adaptive Management

Companion repository folder for the article **“Learning, Memory, and Adaptive Management.”**

This directory provides reproducible workflows for modeling learning, memory retention, forgetting pressure, feedback use, monitoring quality, governance capacity, community knowledge, justice protection, and adaptive response under repeated disturbance.

## Correct article directory

```text
articles/learning-memory-and-adaptive-management/
```

## Purpose

This scaffold supports a professional learning-and-adaptive-management workflow:

1. Define monitoring quality, memory retention, feedback use, governance flexibility, community knowledge, justice protection, implementation reliability, and forgetting pressure.
2. Compare adaptive-management strategies under balanced, monitoring-first, memory-first, feedback-use-first, governance-first, community-knowledge-first, justice-first, and forgetting-sensitive priorities.
3. Simulate how memory, learning, feedback, and governance affect system function under repeated disturbance.
4. Estimate justice-weighted adaptive response.
5. Model uncertainty using Monte Carlo simulation.
6. Diagnose shallow learning, institutional amnesia, weak feedback use, and maladaptive response.
7. Support responsible interpretation across ecological systems, public institutions, infrastructure, climate adaptation, public health, communities, organizations, and social-ecological systems.
8. Provide multi-language examples in Python, R, Julia, SQL, Rust, Go, C, C++, and Fortran.
9. Export reproducible tables, figures, documentation, and notebook placeholders.

## Responsible-use note

The data are synthetic and the models are methodological examples. They are intended for learning, reproducible article companions, dashboard prototyping, and professional modeling design. Real adaptive-management assessment requires empirical monitoring, system history, community participation, governance review, ecological and technical expertise, uncertainty analysis, and accountable follow-through.

Do not use these examples to claim exact adaptive capacity, rank communities, justify experimentation without consent, or replace field validation and participatory governance with model output.

## Quick start

Run the dependency-light Python workflow:

```bash
python3 python/learning_memory_standard.py
```

Optional advanced Python workflow:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-advanced.txt
python python/learning_memory_advanced.py
```

Run smoke checks:

```bash
bash smoke-test.sh
```
