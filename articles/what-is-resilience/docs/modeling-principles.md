# Modeling Principles

This companion directory treats resilience as a system property under disturbance, not as a single static score.

## Core modeling ideas

1. **Disturbance load** changes over time.
2. **Adaptive capacity** modifies response.
3. **Threshold distance** determines how much margin exists before regime change.
4. **Redundancy and modularity** reduce cascading impact.
5. **Learning capacity** can improve response after repeated disturbance.
6. **Exposure and sensitivity** increase risk pressure.
7. **Viability** is interpreted as preserving essential system function above a threshold.

## Ethical cautions

- Do not treat resilience as automatically desirable.
- Do not use resilience language to normalize abandonment or underinvestment.
- Do not reduce community resilience to self-reliance.
- Do not ignore distribution, power, history, or ecological limits.
- Always ask who benefits, who bears risk, and whose system function is being protected.
