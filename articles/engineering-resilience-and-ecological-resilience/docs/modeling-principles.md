# Modeling Principles

This companion scaffold distinguishes engineering resilience from ecological resilience.

## Engineering resilience variables

- return speed
- reliability
- repair capacity
- backup capacity
- service continuity
- recovery time
- uptime
- failover capacity

## Ecological resilience variables

- threshold distance
- basin width
- adaptive capacity
- functional diversity
- redundancy
- modularity
- disturbance exposure
- regime-shift sensitivity

## Modeling logic

Engineering resilience is modeled as recovery-oriented performance. Ecological resilience is modeled as threshold-oriented persistence. The scaffold also calculates an imbalance score to identify systems that may recover quickly but remain close to regime-shift risk, or systems that preserve adaptive capacity despite slower operational return.

## Appropriate use

Use the scaffold for education, synthetic-data modeling, dashboard prototyping, and professional comparative analysis design. Replace synthetic values with validated system-specific data before any real-world use.
