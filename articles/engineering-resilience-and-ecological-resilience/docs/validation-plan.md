# Validation Plan

A professional engineering/ecological resilience model should include:

1. **System boundary validation**  
   Define whether the model is describing a component, network, ecosystem, institution, community, or coupled social-ecological system.

2. **Metric validation**  
   Verify whether engineering metrics and ecological metrics are measuring comparable or fundamentally different properties.

3. **Scenario validation**  
   Test the model across short operational outages, repeated disturbances, compound shocks, and slow threshold erosion.

4. **Uncertainty review**  
   Identify which thresholds are uncertain, which variables are proxies, and which relationships are assumed rather than observed.

5. **Expert review**  
   Engineering systems require technical review. Ecological systems require domain expertise. Social systems require affected-community interpretation and governance review.

6. **Ethical review**  
   Ensure resilience language is not used to normalize underinvestment, burden shifting, or return to harmful conditions.
