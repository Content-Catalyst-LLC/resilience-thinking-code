# Resilience or Abandonment?

Companion repository folder for the article **Resilience or Abandonment?**

This scaffold supports reproducible examples for distinguishing support-oriented resilience from abandonment risk. It focuses on protective effectiveness, material support, accessible recovery, governance inclusion, transformation potential, exposure reduction, burden shifting, implementation burden, abandonment risk, and justice-adjusted resilience.

## Directory

```text
articles/resilience-or-abandonment/
```

## What this scaffold includes

- Synthetic resilience-or-abandonment strategy data.
- Synthetic compound-stress pathway data.
- Dependency-light Python workflow for support-oriented resilience scoring, abandonment-risk diagnostics, and pathway simulation.
- Optional advanced Python workflow using pandas, numpy, and matplotlib.
- R workflow for comparing strategies across support, recovery, governance, transformation, exposure, and burden-sensitive priorities.
- SQL schema for strategy data, pathway data, and value views.
- Lightweight examples in Julia, C, C++, Go, Rust, and Fortran.
- Responsible-use documentation, validation notes, and WordPress GitHub embed block.

## Responsible-use note

These workflows use synthetic data. They are for methods demonstration, reproducible article companions, planning prototypes, and learning. Real resilience-or-abandonment analysis requires affected-community participation, local knowledge, disaggregated vulnerability evidence, public finance review, infrastructure and service data, housing and health analysis, rights review where relevant, and accountable governance.

Do not use these examples as automated policy, emergency, relocation, infrastructure, insurance, aid-allocation, or public-service decision systems.
